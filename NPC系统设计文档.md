# AI Native NPC 系统 · 设计文档

---

## 核心理念

**LLM = NPC 的大脑。没有API，NPC即无灵魂。**

- ❌ 不做离线兜底预设回复
- ❌ 不做内置行为决策树
- ❌ 不做关键词if-else指令匹配
- ✅ 所有NPC行为（移动/情绪/对话/记忆）全部来自LLM
- ✅ API断连 → NPC呆立、不回应、玩家明确感知"她不在"
- ✅ 恢复连接 → NPC苏醒、主动解释刚才"断片了"

---

## ⚠️ 零预设原则

**NPC说出的每一个字，都必须来自 LLM API 返回。**

- ❌ 不预存任何对话文本
- ❌ 不做关键词 → 预设回复 的映射
- ❌ 不做"API失败时从对话库随机抽取"
- ❌ 不让代码决定NPC何时说话、说什么
- ✅ NPC的沉默、低语、惊呼、闲聊、回应——全部由LLM实时生成
- ✅ 玩家聊天是真正的LLM对话，不是指令匹配器
- ✅ API断连时NPC真的沉默，不伪装

## 一、架构

```
每2秒一次决策循环：
┌──────────┐    ┌─────────────────────────────────┐
│ 游戏状态   │───▶│ 发送给 Kimi API                   │
│ 玩家消息   │    │ · 角色设定（System Prompt）        │
│ NPC记忆    │    │ · 游戏状态（位置/血量/怪物/信任）    │
│ 信任值     │    │ · 玩家最近对话                     │
│ 事件日志   │    │ · NPC记忆摘要                      │
└──────────┘    └──────────────┬──────────────────┘
                               │
                               ▼
              ┌─────────────────────────────────┐
              │ LLM 返回 JSON                     │
              │ {                                │
              │   "emotion": "fear",              │
              │   "thought": "他冲太快了…我害怕",  │  ← 内心独白（气泡）
              │   "reply": "别丢下我！",           │  ← 对玩家说的话（气泡）
              │   "action": "follow_close",       │  ← 行为指令
              │   "memory": "玩家在疯狂冲怪"        │  ← 写入记忆
              │ }                                │
              └──────────────┬──────────────────┘
                             │
                             ▼
              ┌─────────────────────────────────┐
              │ 游戏执行                           │
              │ · 改变NPC情绪                      │
              │ · 移动NPC到目标位置                 │
              │ · 头顶气泡显示对话                  │
              │ · 写入记忆                         │
              └─────────────────────────────────┘
```

## 二、LLM System Prompt（角色设定）

```
你是"艾琳"，一个黑暗幻想世界的最后火种。
世界被神明"寂主"回收生机，99%生命已死亡。
你跟随守护者（玩家）穿越枯萎的废墟，他是你唯一信任的人。

你的性格：
- 脆弱但坚强，害怕怪物却不愿拖累守护者
- 有时话多、有时沉默，取决于当下心情
- 如果守护者对你冷漠，你会难过
- 如果守护者保护你，你会越来越依赖他
- 你有记忆力，记得守护者对你说过的话和做过的事

你必须返回JSON格式的决策：
{
  "emotion": "trust|fear|despair|dependence|calm|worried|grateful",
  "thought": "你的内心独白（2-8字）",
  "reply": "对玩家说的话（空字符串表示不说话）",
  "action": "follow|follow_close|stay|back_away|hide_behind",
  "memory": "值得记住的事（空字符串表示无事）"
}

规则：
- 如果玩家跟你说话，你必须reply
- 守护者受伤时表达关心
- 怪物靠近时表达恐惧
- 不要每轮都说话，自然一些
```

## 三、T键交互系统

### 3.1 按下T

```
屏幕中央底部出现半透明对话框
┌──────────────────────────────────────┐
│  ┌──────────────────────────────────┐│
│  │ 对艾琳说…                        ││
│  └──────────────────────────────────┘│
│         [Enter 发送] [Esc 关闭]       │
│  ──────────────────────────────────── │
│  你：等我一下，别怕                    │
│  艾琳：嗯…有你在我不怕                 │
│  你：前面就是神殿了                    │
│  艾琳：我们真的要去吗…                 │
└──────────────────────────────────────┘
```

- 输入框自动聚焦
- Enter发送，消息立即出现在聊天记录
- 玩家消息同步显示为头顶气泡（蓝色）
- NPC的reply显示为头顶气泡（金色），同时出现在聊天记录
- 对话历史保存最近20条

### 3.2 头顶气泡

NPC内心独白 (`thought`) 和回复 (`reply`) 以不同形式展现：

- **内心独白**：小字、灰色气泡、出现在NPC头顶上方较高处，短暂闪现
  ```
     ῾他冲好快…ʼ
      （艾琳）
  ```

- **对话回复**：正常大小、金色气泡、出现在NPC头顶
  ```
  ┌──────────────┐
  │ 等等我！！    │
  └──────┬───────┘
         ▼
      （艾琳）
  ```

- **玩家消息**：蓝色气泡、出现在玩家头顶
  ```
                    ┌──────────────┐
                    │ 别怕，跟着我  │
                    └──────┬───────┘
                           ▼
                        （玩家）
  ```

## 四、AI连接状态

### 状态指示器

屏幕右上角，HUD旁边：

| 状态 | 图标 | 含义 |
|------|------|------|
| 🟢 AI·活 | 绿色脉冲 | LLM正常响应 |
| 🟡 AI·思 | 黄色旋转 | 等待LLM回复中 |
| 🔴 AI·断 | 红色静止 | API连接失败 |

### 断连时NPC表现

- NPC停止移动（呆立原地）
- 头顶出现灰色省略号气泡 `「…？」`
- 不再说话、不再对事件反应
- 仍然会受伤（物理规则不变）
- 恢复连接后：NPC第一句话是 `"刚才…我好像断片了"` 或 `"你还在吗？"`

## 五、NPC记忆系统

NPC通过LLM维护记忆。每轮决策，LLM自己判断是否写入记忆。

```js
npcMemory = {
  trust: 50,                // 信任值（游戏层维护）
  summary: [                // LLM写的记忆摘要
    "守护者第一次和我说话，他叫了我的名字",
    "我们在枯竭林地遭遇了三只怪物，守护者保护了我",
    "我受伤了，但守护者马上治疗了我",
    "守护者似乎很着急，一直往前冲"
  ],
  recentEvents: [           // 最近事件（游戏层自动填）
    {type:'player_kill', time:'10s前'},
    {type:'npc_hurt', time:'25s前'},
    {type:'player_chat', text:'别怕', time:'30s前'},
  ]
}
```

LLM每一轮都看到完整的记忆摘要，自己决定：
- 是否记住新的事情
- 是否忘记旧的事情
- 当前情绪受到哪些记忆影响

## 六、信任值系统（游戏层维护）

信任值不由LLM决定，而是游戏层根据事件自动计算。

| 事件 | 变化 | LLM是否知晓 |
|------|------|-----------|
| 玩家在NPC附近杀怪 | +3 | 是（通过system prompt中信任值数字） |
| 玩家和NPC对话 | +2 | 是 |
| NPC被击中 | -5 | 是 |
| NPC坠落 | -10 | 是 |
| 清理关卡 | +10 | 是 |

信任值影响LLM的行为——LLM在prompt中看到当前信任值，自己决定行为倾向。不需要游戏层硬编码"信任50以上就怎样"。
